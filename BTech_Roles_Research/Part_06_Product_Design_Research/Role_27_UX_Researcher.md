## Overview

A UX Researcher studies user behavior, needs, and pain points through qualitative and quantitative research methods to inform product design and strategy. Unlike UX/UI Designers who create interfaces, UX Researchers focus on understanding users through interviews, surveys, usability testing, and data analysis. This role is less common than UX/UI design and often requires either specialized education (Psychology, HCI, Design) OR significant experience. Fresh B.Tech grads can enter but face more competition than designer roles.

**Key Note for B.Tech Students:** Your analytical and data skills (statistics, SQL, Python) give you an edge in quantitative UX research. However, qualitative research (interviews, ethnography) requires soft skills that may need development.

## Required Skills

### Research Methodologies (Essential)

**Qualitative Methods (Focus: Understanding "Why"):**
- **User Interviews:** 1-on-1 in-depth interviews (30-60 mins)
- **Contextual Inquiry:** Observing users in their environment
- **Focus Groups:** Facilitated group discussions (6-10 participants)
- **Diary Studies:** Users log experiences over days/weeks
- **Ethnographic Research:** Immersive observation (field studies)
- **Usability Testing:** Moderated/unmoderated task-based testing
- **Card Sorting:** Information architecture research
- **A/B Testing (Qual):** User feedback on variants

**Quantitative Methods (Focus: Understanding "What" & "How Much"):**
- **Surveys:** Large-scale questionnaires (100s-1000s of responses)
- **Analytics Analysis:** Google Analytics, Mixpanel, Amplitude
- **A/B Testing (Quant):** Statistical significance testing
- **Eye Tracking:** Heatmaps, gaze patterns
- **Clickstream Analysis:** User navigation patterns
- **Benchmarking Studies:** Comparing metrics to competitors
- **Longitudinal Studies:** Tracking behavior over time

**Mixed Methods:**
- Combining qualitative + quantitative (most common in practice)
- Example: Survey (quant) + follow-up interviews (qual)

### Analysis & Synthesis Skills

**Qualitative Analysis:**
- **Thematic Analysis:** Identifying patterns in interview transcripts
- **Affinity Mapping:** Grouping insights into themes
- **Persona Creation:** User archetypes based on research
- **Journey Mapping:** Visualizing user experience over time
- **Empathy Mapping:** Understanding user thoughts, feelings, actions

**Quantitative Analysis:**
- **Statistical Analysis:** T-tests, chi-square, regression
- **Survey Analysis:** Response rates, cross-tabulation, sentiment analysis
- **Metrics Definition:** DAU, retention, task success rate, SUS (System Usability Scale)
- **Data Visualization:** Charts, graphs, dashboards

### Tools & Software

**Research Platforms:**
- **UserTesting:** Remote unmoderated usability testing
- **Lookback:** Moderated remote testing (video recording)
- **Maze:** Prototype testing, analytics
- **Optimal Workshop:** Card sorting, tree testing
- **UsabilityHub:** Quick 5-second tests, preference tests
- **Dovetail:** Research repository, analysis (tag transcripts, identify themes)

**Survey Tools:**
- **Typeform, Google Forms:** Survey creation
- **Qualtrics:** Advanced survey logic (expensive, enterprise)
- **SurveyMonkey:** Mid-tier surveys

**Analytics:**
- **Google Analytics:** Web/app analytics
- **Mixpanel, Amplitude, Clevertap:** Event tracking, funnels
- **Hotjar, Crazy Egg:** Heatmaps, session recordings
- **Tableau, Looker:** Data visualization

**Collaboration:**
- **Figma, Miro, FigJam:** Journey maps, affinity diagrams
- **Notion, Confluence:** Research documentation
- **Dovetail, Airtable:** Research repository (centralized insights)

**Data Analysis (for B.Tech Students):**
- **Excel/Google Sheets:** Data cleaning, pivot tables
- **SPSS, R, Python:** Statistical analysis (pandas, numpy, scipy)
- **SQL:** Querying product databases

### Soft Skills (Critical for UX Research)

**Empathy & Listening:**
- Active listening without bias
- Reading between the lines (unstated needs)
- Creating safe space for honest feedback

**Communication:**
- Storytelling with data (turn insights into narratives)
- Stakeholder presentations (designers, PMs, leadership)
- Writing research reports (clear, actionable recommendations)

**Facilitation:**
- Running user interviews (asking open-ended questions)
- Moderating usability tests (think-aloud protocol)
- Leading workshops (journey mapping, persona creation)

**Objectivity:**
- Avoiding confirmation bias (not cherry-picking data)
- Representing minority user voices
- Challenging assumptions (including your own)

**Project Management:**
- Managing multiple studies simultaneously
- Recruiting participants (screeners, incentives)
- Timeline management (research takes weeks)

## Day-to-Day Work

### Typical Daily Activities

**Morning (9:00 AM - 12:00 PM):**
- **Check Calendar:** 2 user interviews today (11 AM, 3 PM)
- **Prep for Interview:** Review discussion guide, test recording setup
- **Conduct User Interview (1 hour):**
  - Video call with user (Zoom, Lookback)
  - Ask open-ended questions about pain points
  - Observe user completing tasks in app
  - Record session for later analysis
- **Take Notes:** Document key quotes, observations

**Afternoon (1:00 PM - 5:00 PM):**
- **Analysis (2 hours):**
  - Transcribe interview (Otter.ai, Dovetail)
  - Tag key insights (themes: "checkout confusion", "trust issues")
  - Add quotes to research repository (Dovetail, Notion)
- **Second User Interview (1 hour):**
  - Repeat process
- **Team Sync (30 mins):**
  - Share preliminary findings with PM and designer
  - Discussion: Users struggling with promo code field
  - Decision: Redesign promo code flow (designer action item)

**Evening (5:00 PM - 6:00 PM):**
- **Recruit Participants:** Post screener survey for next week's study (UserTesting, LinkedIn)
- **Stakeholder Update:** Email leadership with research timeline
- **Learning:** Read NNGroup article on "Recruiting Participants"

### Weekly Responsibilities

**Research Execution:**
- Conduct 5-10 user interviews per week
- Run 2-3 usability tests (moderated or unmoderated)
- Send 1-2 surveys (100+ responses)

**Analysis & Synthesis:**
- Review session recordings, transcripts
- Affinity mapping: Group insights into themes
- Create deliverables: Research reports, personas, journey maps

**Collaboration:**
- Design review: Provide research-backed feedback
- Product planning: Share insights to inform roadmap
- Weekly team share-out: Present research findings

**Participant Recruitment:**
- Write screener surveys (filter for target users)
- Schedule sessions, send reminders
- Manage incentives (â‚¹500-1000 per participant)

### Monthly/Quarterly

**Strategic Research:**
- **Generative Research:** Exploratory studies (what should we build?)
- **Evaluative Research:** Validation studies (does this design work?)
- **Benchmarking:** Measure product against competitors
- **Longitudinal Studies:** Track user behavior over 3-6 months

**Deliverables:**
- **Research Reports:** Findings, insights, recommendations
- **Personas:** Updated user archetypes
- **Journey Maps:** End-to-end user experience
- **Opportunity Maps:** Unmet user needs, product gaps
- **Presentations:** Quarterly insights to leadership

**Research Ops (Operations):**
- Maintain research repository (all past studies)
- Update research templates (discussion guides, consent forms)
- Improve recruitment pipeline (panel management)

## Stress Level

**Low to Medium** (lower than PM, engineering, data science)

### Stress Factors

**1. Participant Recruitment Challenges (6/10)**
- Finding qualified participants is time-consuming
- No-shows (participants cancel last-minute)
- Screener design: Filter for right users without leading
- Managing incentives, scheduling across time zones

**2. Stakeholder Skepticism (6/10)**
- "We don't have time for research" (pressure to skip research)
- Convincing leadership to prioritize research over shipping
- Defending findings when they contradict stakeholder opinions
- Balancing rigor vs speed (scrappy research vs academic quality)

**3. Ethical Considerations (5/10)**
- Protecting participant privacy (anonymizing data)
- Avoiding harm (sensitive topics, vulnerable populations)
- Informed consent, data storage compliance

**4. Analysis Overload (6/10)**
- Drowning in qualitative data (10 hours of interviews = 30+ hours analysis)
- Synthesizing conflicting insights (users want different things)
- Pressure to deliver actionable recommendations (not just "interesting findings")

**5. Limited Impact (Sometimes) (5/10)**
- Research findings ignored (team ships design anyway)
- "Research for research's sake" without business impact
- Advocating for users when business goals conflict

### Lower Stress Aspects

âœ… **No Production Pressure:** No on-call, no bugs, no launches
âœ… **Flexible Schedule:** Can work from home (many remote research roles)
âœ… **Intellectual Fulfillment:** Learning about human behavior is fascinating
âœ… **Empathy Work:** Helping users, advocating for better experiences
âœ… **Less Technical Debt:** Unlike engineering, no legacy code to maintain

### Stress Management Tips

1. **Template-ize:** Reuse discussion guides, screeners (don't reinvent each study)
2. **Panel Management:** Build database of willing participants (reduce recruitment time)
3. **Continuous Research:** Small, frequent studies vs giant annual studies
4. **Democratize Research:** Train designers, PMs to do basic research (reduce bottleneck)
5. **Show Business Impact:** Tie findings to metrics (retention, revenue) to increase influence

### Sample Research Projects (For B.Tech Students)

**Project 1: Usability Study**
- **Problem:** College website is hard to navigate
- **Method:** 8 moderated usability tests (task-based)
- **Findings:** 75% task failure on "find faculty contact", confusing information architecture

**Project 2: Survey Study**
- **Problem:** Understand student food delivery app usage

**Project 4: Mixed-Methods**

- **Recommendations:** Iterate promo code placement

**Project 5: Diary Study (Advanced)**
- **Problem:** Understand daily study habits

- **Deliverable:** Journey map, opportunity map

## Technical Preparation

### 1. Learn Research Methods (Qualitative)

**User Interviews (4 weeks):**
- **Learn:** Open-ended questions, active listening, think-aloud protocol
- **Resources:**
  - Book: "The Mom Test" by Rob Fitzpatrick (3 hours read)
  - NNGroup article: "How to Conduct User Interviews"
  - YouTube: "User Interviews 101" (search)
- **Practice:** Interview 10 people (friends, family) about app they use

**Usability Testing (2 weeks):**
- **Learn:** Moderated vs unmoderated, task design, think-aloud
- **Resources:**
  - NNGroup: "Usability Testing 101"
  - UserTesting free trial: Run 3 tests
- **Practice:** Test college website with 5 people, document findings

**Thematic Analysis (3 weeks):**
- **Learn:** Coding transcripts, identifying themes, affinity mapping
- **Resources:**
  - YouTube: "Thematic Analysis Explained"
  - Dovetail Academy (free courses)
- **Practice:** Analyze 10 interview transcripts, create affinity diagram (Miro)

### 2. Learn Quantitative Methods

**Survey Design (3 weeks):**
- **Learn:** Question wording, Likert scales, avoiding bias, sampling
- **Resources:**
  - Coursera: "Survey Data Collection and Analytics" (University of Michigan, free audit)
  - Book: "Designing Surveys" (Coursera course textbook)
- **Practice:** Create 3 surveys (Google Forms), get 100+ responses each

**Statistics (4-6 weeks):**
- **Learn:** Descriptive stats (mean, median, mode), inferential stats (T-tests, chi-square)
- **Resources:**
  - Khan Academy: Statistics (free, comprehensive)
  - Coursera: "Basic Statistics" (University of Amsterdam)
- **Practice:** Analyze survey data in Excel, run T-tests, interpret results

**SQL (4 weeks, Critical for Quantitative Research):**
- **Learn:** SELECT, WHERE, JOINs, GROUP BY, aggregations
- **Resources:**
  - Mode Analytics SQL tutorial (free, product-focused)
  - StrataScratch: Practice SQL questions
- **Practice:** Query sample datasets (user behavior, transactions)

**Python for Data Analysis (Optional, 6 weeks):**
- **Learn:** Pandas (data manipulation), Matplotlib (visualization)
- **Resources:**
  - Kaggle: Python micro-courses (free)
  - freeCodeCamp: Data Analysis with Python
- **Advantage:** Analyze large datasets (clickstream, logs), advanced visualizations

### 3. Learn Tools

**Research Platforms (2 weeks each):**
- **UserTesting:** Free trial, run remote usability test
- **Maze:** Prototype testing, analytics
- **Dovetail:** Tagging insights, affinity mapping (free tier)
- **Miro/FigJam:** Journey maps, affinity diagrams

**Analytics (3 weeks):**
- **Google Analytics:** Free course (Google Skillshop, 8-10 hours)
- **Mixpanel:** Free tier, track events in demo project
- Understand: Funnels, cohorts, retention, drop-offs

**Survey Tools:**
- **Google Forms:** Free, easy surveys
- **Typeform:** Beautiful surveys (free tier limited)

### 4. Soft Skills

**Storytelling with Data (Ongoing):**
- Learn: How to present findings compellingly
- Resources:
  - Book: "Storytelling with Data" by Cole Nussbaumer Knaflic
  - NNGroup: "Presenting UX Research Findings"
- Practice: Present research findings to friends (get feedback)

**Active Listening (Practice Daily):**
- In conversations: Don't interrupt, ask follow-up questions
- Practice: Record yourself in mock interviews, identify leading questions
